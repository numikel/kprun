# Changelog v0.1.1

All notable changes to kprun are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.1.1] - 2026-06-23

### Added

- Two-phase release workflow: local `/prepare-release` Cursor skill plus tag-triggered CI validation and GitHub Release publish
- Keep a Changelog files at `CHANGELOG.md` and `docs/changelogs/vX.Y.Z.md` (release body sourced from per-version file)
- Open-source repo hygiene: MIT LICENSE, SECURITY.md, Contributor Covenant, Dependabot (Cargo + Actions), issue/PR templates, and CODEOWNERS
- CI `cargo audit` (RustSec) and PR `release-prep-check` gate when workspace version changes
- Release workflow validate job: requires matching changelog file and `Cargo.toml` version before building assets
- Maintainer guide at `docs/github-setup.md` (branch protection, Dependabot alerts, first release steps)

### Fixed

- Prefer `KPRUN_TEST_MASTER` over OS keyring in unlock fallback so integration tests stay deterministic on machines with a stored master password

### Changed

- Bump GitHub Actions: `actions/checkout` v7, `actions/download-artifact` v8, `actions/upload-artifact` v7, `softprops/action-gh-release` v3
