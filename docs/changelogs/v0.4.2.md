# Changelog v0.4.2

All notable changes to kprun are documented in this file.

The format is based on [Keep a Changelog](https://keepachangelog.com/en/1.1.0/),
and this project adheres to [Semantic Versioning](https://semver.org/spec/v2.0.0.html).

## [0.4.2] - 2026-07-11

### Changed

- MCP bridge transport now routes through `McpTransportImpl` trait for cleaner
  abstraction between Streamable and legacy SSE transports.
- SSE line buffer reused across events to reduce per-event allocations and improve
  throughput.
- Documented legacy POST body forwarding behavior and Context7 header expectations
  for transport detection.

### Fixed

- MCP bridge shutdown now bounded by fixed timeout to prevent indefinite waits on
  hung server streams.
- Server-stream thread shutdown properly awaited to ensure clean termination on
  session close.
- Connection reset errors in test mock server properly treated as end-of-stream
  instead of catastrophic failure.
- Notifications and `initialized` messages replayed on session re-initialization
  to restore coherent MCP state.
- Response body timeout exemptions now applied consistently across all transport types.
- MCP-Protocol-Version header sent on GET stream initiation and DELETE shutdown
  for proper version negotiation.
